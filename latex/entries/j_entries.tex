\dictentry{Java Card}{/ˈdʒɑːvə kɑːrd/, \textit{n.}}{
A technology that allows small Java-based applications (applets) to run
securely on smart cards and similar small-memory devices. It enables a single
card to hold multiple different applications, such as a credit card applet and
a transit pass applet, on the same chip.

\example{Many modern EMV credit cards use Java Card technology, which allows the
bank to securely update or add new features to the card's chip even after it has
been issued.}

\analogy{Biological}{Like \textbf{Plasmids} in bacteria. Just as a bacterium
can carry small, independent loops of DNA called plasmids that give it extra
"abilities" (like antibiotic resistance) without changing its main genome, a
smart card can carry Java Card applets that give it specific "abilities" (like
processing a Visa or Mastercard payment) independent of the card's main
operating system.}

\crossref{\textbf{Smart Card}, \textbf{OpenCard Framework (OCF)}, \textbf{EMV}}

\etymology{Introduced by Sun Microsystems in 1996 to bring the portability of
Java to the highly constrained environment of smart card chips.}
}
