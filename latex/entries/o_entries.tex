\dictentry{OpenCard Framework (OCF)}{/ˈoʊpən kɑːrd ˈfreɪmwɝːk/, \textit{n.}}{
A legacy, Java-based middleware framework that provides a standardized,
platform-independent interface for smart card applications. Developed in the
late 1990s, OCF allowed Java programs to communicate with various card readers
and smart cards using a high-level API, shielding developers from the
complexities of different hardware drivers.

\example{A bank building a Java-based teller application in 2002 might have used
the OpenCard Framework to ensure their software could read customer chip cards
using any brand of card reader.}

\analogy{Everyday}{Like a \textbf{Universal Remote Control} for home
electronics. Instead of needing a different remote for your TV, DVD player, and
sound system, a universal remote provides a single set of buttons that work
regardless of the manufacturer. OCF did the same for smart cards, providing a
single "remote" for Java developers to talk to any card reader.}

\crossref{\textbf{APDU}, \textbf{ISO/IEC 7816}, \textbf{Smart Card}}

\etymology{Created by the OpenCard Consortium, led by IBM and Gemplus, to bring
Java's "Write Once, Run Anywhere" philosophy to the smart card industry.}
}
